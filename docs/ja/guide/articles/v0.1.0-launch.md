# v0.1.0 公開ウォークスルー

![v0.1.0 release header](/release-header-v0.1.0.svg)

`v0.1.0` は、`onizuka-openclaw-autonomous-team-starter` を外へ出せる最初の形にまとめたリリースです。Windows 前提の Podman 運用、persona scaffold、Mattermost 会話ラボを 1 つの starter に束ねています。

## このリリースが意味すること

OpenClaw 自体には Podman や provider の説明がありますが、複数 agent を同じローカル環境で再現性高く立ち上げるには glue が要ります。この repo はその glue を、manifest、workspace scaffold、PowerShell entrypoint、Mattermost 導線としてまとめています。

## v0.1.0 で入るもの

### 1 台から triad まで起動できる starter

- `uv run openclaw-podman init`、`launch`、`status`、`logs`、`stop`、`print-env` で基本ライフサイクルを扱えます。
- scale モードでは `.openclaw/instances/agent_*/` に agent ごとの state を生成し、それぞれ独立した config、workspace、port を持たせます。

### 無名 container ではなく teammate として起動できる

- 各 workspace には `AGENTS.md`、`SOUL.md`、`IDENTITY.md`、`USER.md`、`HEARTBEAT.md`、`TOOLS.md`、`BOOTSTRAP.md` が入ります。
- そのまま debate team、writing room、verification squad のような役割分担へ寄せやすいのが、この starter の実務的な価値です。

### Mattermost を同じ operator 導線で扱える

- ローカル Mattermost pod の初期化、bot seed、smoke、lounge enable までを同じ repo から回せます。
- 現在の helper stack は `scripts/mattermost_tools/` にあり、state 読み取りと action を heartbeat-first の小さな tool に分離しています。

### docs と automation まで含めて共有しやすい

- VitePress docs を英語・日本語で公開できます。
- Windows CI は Python compile、unit test、docs build、CLI help、dry-run operator flow を確認します。
- GitHub Pages で release note と walkthrough も同じ docs surface に載せられます。

## 任意の視覚レイヤー

この release には、実験用の Rokuseki Mattermost branding plugin bundle と UI 検証スクリーンショットも含まれます。base starter を使うだけなら必須ではありませんが、会話ラボをどこまで repo 内で仕上げられるかの実例になります。
